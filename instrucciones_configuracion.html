Cuenta Clara v2 — instrucciones rápidas

Qué cambió en esta versión

  • El visor.html ya no trae la clave de descifrado escrita en el código.
  • El visor pide una clave al abrir. Sin esa clave no puede descifrar el JSON.
  • El editor también pide la clave para cifrar/descifrar. Debe ser la misma clave que usará el visor.
  • Puedes marcar “Recordar clave en este dispositivo”, pero eso baja seguridad si alguien usa el dispositivo desbloqueado.

Archivos incluidos

  • editor.html: para capturar movimientos y guardar el JSON cifrado.
  • visor.html: para consultar el estado de cuenta y calcular el saldo.
  • apps_script_code.gs: código de Google Apps Script para guardar/leer el JSON cifrado en Drive.

Configuración de Apps Script

  1. Abre Google Apps Script.
  2. Crea un proyecto o abre el existente.
  3. Pega el contenido de apps_script_code.gs.
  4. En Configuración del proyecto → Propiedades de secuencia de comandos, configura:
ADMIN_TOKEN = tu_token_largo_para_guardar
FOLDER_ID = solo_el_id_de_tu_carpeta_de_drive
FILE_NAME = estado_prestamos_cifrado.json

FOLDER_ID debe ser solo el ID de la carpeta, no la URL completa.

URL: https://drive.google.com/drive/folders/1ABCxyz...?usp=sharing
FOLDER_ID: 1ABCxyz...
  1. Implementa como App web.
  2. Ejecutar como: Yo.
  3. Quién tiene acceso: Cualquiera.
  4. Copia la URL que termina en /exec.

Configuración del editor

En editor.html, cambia:

const API_URL = 'TU_URL_DE_APPS_SCRIPT/exec';
const ADMIN_TOKEN = 'EL_MISMO_ADMIN_TOKEN_DE_APPS_SCRIPT';

No pongas la clave de cifrado en el código. La capturas en la pantalla del editor.

Configuración del visor

En visor.html, cambia solo:

const API_URL = 'TU_URL_DE_APPS_SCRIPT/exec';

El visor pedirá la clave al abrir. Esa clave debe ser la misma que usaste en el editor al guardar.

Prueba recomendada

  1. Abre el editor.
  2. Captura la clave de cifrado.
  3. Agrega un movimiento positivo y uno negativo.
  4. Guarda en Drive.
  5. Abre el visor.
  6. Captura la misma clave.
  7. Debe mostrar historial y saldo calculado.

Seguridad importante

  • Bien: visor público sin clave escrita en el código.
  • Mal: clave fija dentro del HTML si vas a subirlo a una web pública.
  • Usa una frase larga, no un PIN de 4 dígitos. Ejemplo: PrestamosFamilia2026!
  • Si el navegador con cuentas Google logueadas falla al llamar Apps Script, considera usar Cloudflare Worker como puente.
